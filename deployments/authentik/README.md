# Authentik Configuration-as-Code

Complete CLI-driven Authentik deployment and management system. No GUI required.

## Quick Start

```bash
cd afterdark-darkd/deployments/authentik

# Bootstrap everything (admin user, API tokens, OAuth apps)
make bootstrap

# Check status
make status

# Export current configuration
make export

# List resources
make list-apps
make list-users
make list-flows
```

## Directory Structure

```
afterdark-darkd/deployments/authentik/

```

## CLI Commands

### Setup & Bootstrap

```bash
# Complete bootstrap (run once)
make bootstrap
```

This script:
1. Waits for Authentik to be ready
2. Creates/resets admin user (akadmin / AfterDark2026!)
3. Creates persistent API token
4. Configures OAuth2 provider for AfterDark Security Suite
5. Applies custom blueprints
6. Exports current configuration

### Configuration Management

```bash
# Export configuration to JSON files
make export

# Import configuration (review & apply manually)
make import

# Apply blueprints from blueprints/ directory
make apply-blueprints
```

### Resource Management

```bash
# List all resources
make list-apps       # Applications
make list-providers  # OAuth/SAML providers
make list-flows      # Authentication flows
make list-users      # Users
make list-groups     # Groups

# User management
make create-user     # Interactive user creation
make reset-password  # Reset any user's password
```

### Monitoring & Debugging

```bash
# Check Authentik status
make status

# Test API authentication
make test-auth

# Tail logs
make logs
```

## Python API Client

Direct API access for scripting:

```bash
# Set credentials
export AUTHENTIK_TOKEN="your-token-here"

# Export configuration
python3 scripts/authentik_client.py export --output ./config

# Import configuration
python3 scripts/authentik_client.py import --input ./config

# List specific resources
python3 scripts/authentik_client.py list applications
python3 scripts/authentik_client.py list users

# With username/password auth
python3 scripts/authentik_client.py \
  --username akadmin \
  --password AfterDark2026! \
  export
```

## Blueprints (Infrastructure-as-Code)

Blueprints define Authentik resources declaratively using YAML.

### Example: OAuth2 Provider

```yaml
version: 1
metadata:
  name: My OAuth2 App
entries:
  - model: authentik_providers_oauth2.oauth2provider
    identifiers:
      name: My Provider
    attrs:
      client_id: my-client-id
      redirect_uris: http://localhost:8080/callback

  - model: authentik_core.application
    identifiers:
      slug: my-app
    attrs:
      name: My Application
      provider: !KeyOf my-provider
```

### Applying Blueprints

```bash
# Copy blueprint to container
docker cp my-blueprint.yaml authentik-server-prod:/blueprints/custom/

# Apply via CLI
make apply-blueprints

# Or via Django management command
docker exec authentik-server-prod \
  python -m manage blueprint_import /blueprints/custom/my-blueprint.yaml
```

## Environment Variables

Configuration in `.env` (auto-generated by bootstrap):

```bash
AUTHENTIK_TOKEN=<api-token>          # API authentication token
AUTHENTIK_URL=http://localhost:9000  # Authentik URL
AUTHENTIK_ADMIN_USER=akadmin         # Admin username
```

Override defaults:

```bash
export AUTHENTIK_URL=https://auth.afterdark.local
export AUTHENTIK_ADMIN_PASSWORD=MySecurePassword
make bootstrap
```

## Configuration Workflow

### Initial Setup

```bash
# 1. Bootstrap Authentik
make bootstrap

# 2. Export baseline configuration
make export

# 3. Commit configuration to git
git add config/ blueprints/
git commit -m "Add Authentik baseline configuration"
```

### Making Changes

```bash
# 1. Make changes via CLI or API (not GUI)
python3 scripts/authentik_client.py ...

# 2. Export updated configuration
make export

# 3. Review changes
git diff config/

# 4. Commit changes
git add config/
git commit -m "Update OAuth scopes for AfterDark app"
```

### Deploying to New Environment

```bash
# 1. Bootstrap fresh instance
make bootstrap

# 2. Apply custom blueprints
make apply-blueprints

# 3. Verify configuration
make status
make test-auth
```

## Backup & Restore

### Backup

```bash
# Export configuration
make export

# Backup database (if using external PostgreSQL)
pg_dump authentik > authentik_backup.sql

# Or use provided backup script
../../scripts/backup_authentik.sh
```

### Restore

```bash
# Restore database
psql authentik < authentik_backup.sql

# Re-apply blueprints
make apply-blueprints
```

## Troubleshooting

### Authentication Failed

```bash
# Check token
cat .env | grep AUTHENTIK_TOKEN

# Test authentication
make test-auth

# Re-create token
make bootstrap
```

### Container Not Running

```bash
# Check container status
docker ps --filter "name=authentik"

# View logs
make logs

# Restart container
docker restart authentik-server-prod
```

### Blueprint Errors

```bash
# Check blueprint syntax
yamllint blueprints/*.yaml

# View Authentik logs during import
docker logs -f authentik-server-prod 2>&1 | grep -i blueprint

# Validate blueprint manually
docker exec authentik-server-prod \
  python -m manage blueprint_validate /blueprints/custom/my-blueprint.yaml
```

## Production Deployment

### Security Checklist

- [ ] Change default admin password
- [ ] Use external PostgreSQL (not SQLite)
- [ ] Enable HTTPS/TLS
- [ ] Set secure `SECRET_KEY`
- [ ] Configure email for password reset
- [ ] Enable backup automation
- [ ] Set up monitoring/alerting
- [ ] Restrict API token permissions
- [ ] Enable audit logging
- [ ] Configure session timeouts

### Production Bootstrap

```bash
# Set production values
export AUTHENTIK_URL=https://auth.afterdark.com
export AUTHENTIK_ADMIN_PASSWORD="$(openssl rand -base64 32)"
export AUTHENTIK_ADMIN_EMAIL=admin@afterdark.com

# Bootstrap with production settings
make bootstrap

# Save credentials to secure vault
echo "$AUTHENTIK_ADMIN_PASSWORD" | vault write secret/authentik/admin password=-
```
